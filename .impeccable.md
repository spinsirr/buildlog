## Design Context

### Users
Dev teams who ship every day and need that work to drive distribution. They connect GitHub once and get AI-generated social media posts from their commits, PRs, and releases. They're technical, opinionated about design, time-constrained, and want the tool to feel like it was built by one of them. Context: quick dashboard checks between coding sessions — scan posts, publish, move on.

### Brand Personality
**Bold. Developer-first. Effortless.**

The landing page is loud and unapologetic — neo-brutalism with thick borders, drop shadows, and bright accents. The dashboard is the opposite: dark, minimal, fast. This split is intentional: the landing sells energy, the dashboard delivers calm efficiency.

### Aesthetic Direction
- **Landing**: Neo-brutalism — cream (#FFFDF5) background, 2-4px black borders, 4-8px black box-shadows, bold typography (Space Grotesk), bright accent palette (red #FF6B6B, yellow #FFD93D, lime #BFFF00, purple #C4B5FD). Slight card rotations (1-3deg). Playful, high-energy, 90s-meets-AI.
- **Dashboard**: Dark minimal — zinc-950 background, zinc-900 cards, thin zinc-800 borders, Geist sans-serif, IBM Plex Mono for secondary text. Clean, professional, zero noise. Purple-400 accent on "Log" wordmark.
- **Anti-references**: Corporate SaaS (Salesforce energy), overly rounded/pastel designs, Material Design blandness, cluttered dashboards with too many metrics.
- **References**: Linear (dark UI clarity), Raycast (developer speed), cal.com (clean dashboard), Vercel dashboard (dark + purposeful).

### Design Principles
1. **Density over decoration** — Show more content, less chrome. Every pixel of UI should earn its space.
2. **Two speeds, one brand** — Landing is loud, dashboard is quiet, but they share DNA (the 4-color logo palette, monospace touches, sharp corners).
3. **Keyboard-first, glance-friendly** — Dashboard interactions should feel instant. Status should be readable in <1 second.
4. **Mono as a weapon** — Monospace isn't just for code. Use IBM Plex Mono for timestamps, counts, labels, badges — it signals "developer tool" without being noisy.
5. **Motion with purpose** — Animate state changes (publish, connect, error), not decoration. No bouncing logos or gratuitous parallax.

### Typography
```
--font-sans: Geist (body, UI)
--font-display: Space Grotesk 700 (landing headlines)
--font-mono-ui: IBM Plex Mono 400/700 (labels, timestamps, badges)
--font-code: JetBrains Mono 400/700 (code blocks)
```
